Файловая система

Файловая система (ФС) — это механизм для управления данными на устройствах хранения (жестких дисках, SSD, флеш-накопителях и других).

Основные функции файловой системы:

  • Организация хранения данных - структурирование информации на носителе
  • Управление метаданными файлов - хранение информации о файлах (имя, размер, дата)
  • Обеспечение доступа к данным - чтение, запись, удаление файлов
  • Управление правами доступа - контроль доступа пользователей к файлам
  • Восстановление после сбоев - обеспечение целостности данных

Типы файловых систем:

  • FAT16/FAT32 - File Allocation Table (флешки, карты памяти, старые Windows)
  • NTFS - New Technology File System (современные Windows)
  • ext3/ext4 - Extended File System (Linux)
  • HFS+/APFS - Hierarchical File System (macOS)
  • exFAT - Extended FAT (флеш-накопители больших объемов)

Механизм ввода-вывода

Механизм ввода-вывода отвечает за взаимодействие операционной системы с внешними устройствами.

Типы устройств ввода-вывода:

  • Блоковые устройства - работают с данными блоками фиксированного размера (HDD, SSD, CD/DVD)
  • Потоковые устройства - обрабатывают данные по одному элементу (клавиатура, мышь, принтер)
  • Сетевые устройства - обеспечивают сетевое взаимодействие (сетевые карты, модемы)

Методы доступа к устройствам:

  • Программный ввод-вывод - CPU непосредственно управляет устройством
  • Прерывания - устройство уведомляет CPU о готовности
  • Прямой доступ к памяти (DMA) - данные передаются без участия CPU

Архивация данных

Архивация данных — это процесс сжатия и хранения данных в одном или нескольких файлах для экономии пространства.

Методы архивации:

  • Без потерь - ZIP, GZIP, RAR, 7z (данные восстанавливаются полностью)
  • С потерями - JPEG, MP3, MPEG (часть данных теряется для большего сжатия)

Алгоритмы сжатия:

  • LZ77/LZ78 - словарные методы сжатия
  • Huffman coding - статистическое кодирование
  • Run-length encoding (RLE) - кодирование серий
  • BWT - преобразование Барроуза-Уилера
